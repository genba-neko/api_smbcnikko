# プラン: 汎用 D1 スキーマ管理ツール

## 目的

`schemas/` 配下の SQL ファイルを読み込み、Cloudflare D1 に対してテーブルの作成・削除・再作成を行う汎用シェルスクリプトを作成する。
既存の `tools/webauthn_sign_count.sh`（import 機能付き）は残し、create/drop/recreate は新スクリプトに集約する。

## 関連資料

- `schemas/webauthn-sign-count.sql`
- `schemas/yuutai.sql`
- `tools/webauthn_sign_count.sh`（参考実装）

## インターフェース

```bash
# 特定スキーマを操作
./tools/d1_schema.sh create    webauthn-sign-count
./tools/d1_schema.sh drop      yuutai
./tools/d1_schema.sh recreate  webauthn-sign-count

# 全スキーマを一括操作（--all を明示必須）
./tools/d1_schema.sh create   --all
./tools/d1_schema.sh drop     --all
./tools/d1_schema.sh recreate --all

# dry-run（実行せず生成 SQL を標準出力に表示）
./tools/d1_schema.sh drop     yuutai    --dry-run
./tools/d1_schema.sh recreate --all     --dry-run
```

- 第2引数: スキーマ名のみ受け付ける（`schemas/<name>.sql` を参照）
- `--all` を使う場合は第2引数を省略し `--all` を指定する
- `--dry-run` は任意の操作に付加可能

### 環境変数

| 変数 | デフォルト | 説明 |
|---|---|---|
| `DATABASE` | `smbcnikko-db` | D1 データベース名 |
| `TARGET` | `remote` | `remote` または `local` |

## ファイル構成

```
schemas/
├── webauthn-sign-count.sql       # CREATE TABLE / CREATE INDEX
├── webauthn-sign-count_drop.sql  # DROP INDEX / DROP TABLE（対ファイル）
├── yuutai.sql
└── yuutai_drop.sql
```

`_drop.sql` を対で管理する理由:
- シェルによる SQL パースは改行・コメント・将来の構文追加で壊れるリスクがある（Codex レビュー指摘）
- DROP の実行順序（INDEX → TABLE）を明示的にコントロールできる
- 既存の `webauthn_sign_count.sh` での `drop.sql` 管理パターンと一致

## 実装方針

### create

`schemas/<name>.sql` を `npx wrangler d1 execute --file=` で実行。

### drop

`schemas/<name>_drop.sql` を `npx wrangler d1 execute --file=` で実行。
`_drop.sql` が存在しない場合はエラーで終了。

### recreate

`drop` → `create` の順に実行。drop 失敗時は create を実行しない（`set -euo pipefail` で保証）。
`--dry-run` 時は両方の SQL を標準出力に出力するのみ。

### --all 時の実行順序

`schemas/*.sql`（`_drop.sql` 除く）をアルファベット順に処理。
現状 FK 依存はないが、将来的にはファイル名プレフィックス（`01_`, `02_`）で順序制御する方針とする。

### 一時ファイル

`data/tmp/` 以下に生成し、`trap` で終了時に削除。
（既存スクリプトの `run_d1_file` がリポジトリルート相対パス変換を行うため、`/tmp/` は使わない）

## 方針メモ

- `set -euo pipefail` で堅牢に
- `data/tmp/` の一時ファイルは `trap` で削除

- D1 実行は全て既存の `run_d1_file` パターンに倣う
